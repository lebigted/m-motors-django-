STRUCTURE — m-motors-django (Backend API)
==========================================

Stack : Django 4.x · Django REST Framework · SimpleJWT · pytest


APPLICATIONS
------------

accounts/
  Modèle utilisateur custom (email comme identifiant, rôle admin/client).
  Auth JWT : access token 8h, refresh token 7 jours.
  Vues : login, register, profil (GET/PATCH), liste clients (admin).

vehicles/
  Modèle Vehicle : type achat/location, services LLD, options, photo.
  CRUD complet côté admin, lecture publique sans auth.
  Actions : toggle achat ↔ location, changement de statut.

dossiers/
  Trois modèles dans dossiers/models/ :
    dossier.py   Demande client, 10 statuts, documents joints, messages.
    contrat.py   Cycle de vie complet (9 statuts : a_signer → actif → termine).
    sav.py       Tickets SAV et leurs messages.

  Même découpage dans serializers/ et views/ (un fichier par domaine).


CONFIGURATION
-------------

mmotors/
  settings.py   Configuration projet (JWT, CORS, media, logging, Sentry).
  urls.py       Routage global.
  wsgi.py       Point d'entrée WSGI.


FICHIERS UTILES
---------------

.github/
